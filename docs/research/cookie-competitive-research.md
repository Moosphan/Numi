# Cookie 记账竞品调研摘要

调研日期：2026-06-18  
目标：为 Numi 纯本地记账 App 的 PRD、信息架构和 UI 组件库提供依据。

## 1. 基本信息

Cookie 记账是一款中国区 iOS 财务类 App，由 Beijing Noncurious Technology Co., Ltd. 开发。App Store 信息显示，截至调研时当前版本为 2.27.3，发布日期为 2026-06-02，评分约 4.88，评价数量 15976。官方卖点包括无广告、无开屏、无理财、AI 记账、多账本、预算、存钱、订阅分期、多币种、iCloud 同步、小组件和隐私锁。

## 2. 核心体验判断

Cookie 的竞争力不只是“功能多”，而是把高频记账做得很轻：

- 主界面围绕“明细、趋势、我的”三页展开。
- “记一笔”始终作为悬浮主操作出现。
- 记账页以分类图标网格加金额键盘为中心，用户可以快速完成分类和金额输入。
- 趋势页用浅色卡片和列表进度条表达财务状态，降低专业图表压力。
- 低频能力如预算、订阅、分期、多账本、报销、主题等被放在更深层入口，避免干扰日常记账。

## 3. 视觉风格

可借鉴原则：

- 浅色页面、大面积留白、轻卡片、少边框。
- 柔和语义色表达支出、收入、结余、记录次数。
- Emoji 图标让分类更轻松，但需要提供紧凑和无图标模式。
- 金额和分类层级清晰，备注、时间、账户弱化。
- 图表不过度复杂，优先“看一眼知道花在哪里”。

需要规避：

- 不复制 Cookie 的品牌表达、截图布局和具体资产。
- 不把 UI 做成单一色系主题。
- 不让可爱感牺牲财务数据的严谨和可访问性。

## 4. 用户评价洞察

正向反馈集中在：

- 简洁、好看、无广告。
- 容易坚持记账。
- 功能丰富但上手不重。
- 预算、订阅分期、小组件、快捷指令对习惯养成有帮助。

需求和痛点集中在：

- 预算：周预算、分类周预算、预算顺延、超支后动态日均。
- 退款：希望冲减原支出，而不是算收入。
- 支付：希望一笔订单支持组合支付。
- 资产：希望银行/信用卡子账户、共享额度、多币种账户。
- 循环：希望固定支出、定投等循环规则更灵活。
- 报销：希望可选择是否计入个人预算和收支。
- 趋势：希望模块可隐藏、排序，时间范围更自由。
- 图标：希望可隐藏、缩放、排序和切换风格。
- 数据：希望导入导出、恢复、状态反馈更可靠。

## 5. 对 Numi 的产品机会

Numi 不应只做 Cookie 的本地版，而应建立自己的清晰差异：

- 默认本地保存，无账号，无服务器上传。
- 用户拥有完整数据控制权：JSON、CSV、加密备份、恢复点。
- 首版不依赖云端 AI，后续智能能力采用本地优先和用户主动开启。
- 把 UI 组件库和设计 token 前置，形成可复用的审美系统。
- 强化 Cookie 用户评价中反复出现的复杂场景：退款冲减、组合支付、报销预算规则、周预算、循环规则、子账户。

## 6. 建议优先级

P0：

- 本地数据库。
- 快速记账。
- 明细列表。
- 分类和账户。
- 月概览与基础趋势。
- 月预算/周预算基础版。
- 导入导出。
- 隐私锁和后台模糊。
- 基础 UI 组件库。

P1：

- 订阅、分期、循环记账。
- 报销标记。
- 退款冲减。
- 组合支付。
- 多账本。
- 多币种与手动汇率。

P2：

- 小组件。
- Apple Watch。
- Mac/iPad 大屏优化。
- 本地 OCR。
- 资产趋势和信用卡额度。
